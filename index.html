3. defer attribute

3.1 The defer attribute is used in HTML script tags to defer the execution of a Javascript file until the HTML page has finished parsing.

3.2 This means that the script will only be executed after the HTML has been completed loaded, parsed, and displayed to the user. This is useful for optimizing page load times because it allows the HTML to be displayed to the user faster, and the javascript can execute in the background.

3.3 If you want to add script tag in the head section of the html page, then you should add defer to the script tag as it will be deferred until after the HTML has been fully loaded and parsed.

4.2 Note that "use strict" must be placed at the top of your JavaScript file or function in order to take effect.

8. Ternary Operator

8.1 The ternary operator in javascript is a concise way to write conditional statements. It allows you to evaluate a condition and return one of two expression based on the result of the condition.

8.2 The ternary operator is often used as a shorthand alternative to writing if-else statements. It's useful when you need to assign a value to a variable or return a value based on a condition.

8.3 Using the ternary operator can make your code more concise and readable, especially for simple conditional assignments or returns.

8.4 However, it's important to use it judiciously to maintain code clarity and avoid excessive nesting, which can make the code harder to understand.

9. Switch Statement

9.1 A switch statement in javascript is a control flow statement that allows you to perform different actions based on different conditions.

9.2 It provides a more concise and organized way to write multiple 'if...else' statement.

9.3 The switch statement is a useful tool when you have multiple conditions to check against a single expression. It provides a clearer and more efficient alternative to writing multiple 'if...else' statements.

10. While Loop

10.1 In javascript, a while loop is a control flow statement that allows you to repeatedly execute a block of code as long as a specified condition is true.

10.2 Remember to ensure that the condition within the while loop eventually becomes false; otherwise, the loop will continue indefinitely, resulting in an infinite loop.

10.3 It's important to include a mechanism within the loop to modify the condition and eventually exit the loop.

11. for Loop

11.1 In JavaScript, a for loop is used to repeatedly execute a block of code for a specific number of iterations. It consists of three parts: initialization, condition, and iteration expression.

12. Break keyword

12.1 In JavaScript, the break keyword is used within loops to exit the loop prematurely. It is typically used in conjunction with an if statement to test for a certain condition and then exit the loop if the condition is met.

12.2 The break keyword can also be used with nested loops to exit the innermost loop.

13. Continue keyword

13.1 In Javascript, the continue keyword is used inside loop structures (such as 'for', 'while', or 'do...while') to skip the rest of the current iteration and move on to the next iteration. It is primarily used to control the flow within loops.

13.2 When the continue statement is encountered, the remaining code inside the loop block for the current iteration is skipped, and the loop immediately proceeds to the next iteration.

13.3 The continue statement is handy when you want to execute specific values or conditions from being processed within a loop and proceed to the next iteration.

14. do while loop

14.1 The do while loop is a variation of the while loop. The do-while loop executes a block of code repeatedly until a specified condition evaluates to false.

14.2 The key difference between the do-while and the while-loop is that the do-while loop always executes the block of code at least once, regardless of whether the condition is initially true or false.

14.3 It's important to ensure that the condition within the while statement eventually evaluates to false; otherwise, the loop will continue indefinitely, resulting in an infinite loop.

16. Arrays

16.1 An ordered collection of items or elements is commonly called as an array.

16.2 An array is a data structure that allows you to store multiple values in a single variable. Arrays can hold elements of any data type, including numbers, strings, objects, and even other arrays.

16.4 Arrays are mutable, which means that their elements can be modified after they have been created.

16.5 In javascript, push() and pop() methods are generally faster than the unshift() and shift() methods. The reason for this is that push() and pop() only modify the last element of the array, whereas unshift() and shift() modify the first element of the array, which requires shifting all the remaining elements to new indexes.

17. Primitive v Reference Data Types

17.1 When you assign a primitive data type (such as number) to a variable, the value is copied to the variable.

17.2 This means that changes made to the value of one variable will not affect the value of the another variable that was assigned the same value.

17.3 Arrays are reference data types in javascript. When you assign an array to a variable, the variable actually stores a reference to the array in memory.

17.4 When you assign a reference type to another variable, both variables refer to the same object in memory. Therefore, changes made to the object through one variable are reflected in the other variable as well.

20. Function Declaration

20.1 In javascript, function declarations are used to create named functions that can be called later in the code. A function declaration consists of the 'function' keyword followed by the name of the function, a list of parameters enclosed in parentheses, and the function body enclosed in curly braces.

20.2 It's important to note that function declarations create functions as variables in the current scope. Therefore, they can be accessed and called from anywhere within the scope.

20.3 Function declarations are particularly useful when you want to create reusable pieces of code that can be called multiple times throughout your program.

20.4 Calling a function and invoking a function refer to the same thing - the act of executing a function in a program.

20.5 The purpose of the return statement is to exit the function as soon as the target value is found. Without the 'return' statement, the function would continue iterating through the rest of the array even after the target value is changed.

21. Function Expression

21.1 In javascript, a function expression involves assigning a function to a variable or a constant. Function expressions can be named or anonymous.

21.2 Function expressions provide a way to create functions on the fly, assign them to variables, and use them in various contexts within javascript.

22. Arrow Functions

22.1 Arrow functions also known as fat arrow functions, are a concise syntax introduced in ECMAScript 6 (ES6) for creating functions in Javascript.

22.2 They provide a shorter syntax compared to traditional function expressions and lexically bind this value, resulting in some differences in behavior.

22.3 Arrow functions are used in scenarios where a concise and lexical 'this' binding is desired, such as in event handlers, array methods like map(), filter() and reduce() and when working with a asynchronous functions.

22.4 It's important to note that arrow functions are not suitable for all use cases, particularly when dynamic scoping or the this value is needed. In such cases, traditional function expressions are still the appropriate choice.

22.5 You can remove the parentheses, curly braces and return keyword when using an arrow function in javascript if you have only one parameter to pass.

23. Function Declaration v Function Expression v Arrow Function

23.1 The choice between function declarations, function expressions, and arrow functions depends on the specific context and requirements of your code. Each type of function has its own strengths and use cases.

23.2 Scope and hoisting requirements: If you need the function to be accessible throughout the entire scope and be hoisted, function declarations are suitable choice. Otherwise, function expressions or arrow functions can be used.

23.3 Readability and code style: Arrow functions provide a more concise syntax, which can enhance code readability, especially for simple and short functions. However, for more complex functions, traditional function declarations or expression may be easier to understand.

23.4 'this' binding: Arrow functions have a lexically bound this value, which can be advantageous in certain cases. If you rely on dynamic scoping or need to manipulate the this value, regular functions or function expressions may be more appropriate.

23.5 In summary, there is no single best type of function. Each has its own strengths and use cases, so choose the one that best suits the specific requirements and coding style of your project.

24. Hoisting

24.1 Hoisting is a JavaScript mechanism that allows variables and function declaration to be moved to the top of their respective scopes during the compilation phase.

24.2 Hoisting is a concept specific to variable and function declarations, not their initialization or assignments. The actual assignment or initialization still happens at the original position in the code.

24.3 Variable Hoisting (var, let and const): When variable are hoisted, the variable declaration are moved to the top of their scope while keeping their assignments in place. However, only the declarations are hoisted, not the initializations. If you access a variable before its declaration, it will have the value 'undefined'.

24.4 Function Hoisting: Function declarations are fully hoisted, which means that both the function name and its body are moved to the top of the scope. As a result, you can call a function before its actual declaration in the code.

24.5 Hoisting with Function Expression: Function expressions, which are assigned to variables, are not hoisted in the same way as function declarations. Only the variable declaration is hoisted, not the function assignment.

24.6 It's important to understand hoisting to avoid unexpected behavior and write clear and maintainable code. To ensure code clarity, it is generally recommended to declare variables at the top of their scope and declare functions before they are used, regardless of hoisting behavior.

25. Functions inside function

25.1 When a function is defined inside another function, it is referred to as a "nested function" or "inner function". The outer function is commonly known as the "parent function" or "containing function".

25.2 It's important to note that inner functions are only accessible within the scope of the outer function. They cannot be directly called from outside the outer function. However, if the outer function returns the inner function or exposes it through an object, you can invoke the inner function from the outside.

26. Lexical Scope

26.1 It refers to the scope resolution of variables and functions based on the static structure of the code, specifically the location where the variables and functions are defined in the source code. Lexical scope determines how variable names are resolved during the execution of the code.

26.2 When a variable or function is defined inside another function, the inner function has access to the variables and functions of its outer function. This access is possible because javascript uses lexical scoping. Lexical scoping means that the scope of a variable or function is determined by its position in the source code during the lexical analysis phase, regardless of where it is called during the runtime.

26.3 It is important for understanding how variables are resolved and accessed in javascript. It helps create encapsulation and enables the use of of closures, where functions retain access to variables even after their outer function has finished executing.

27. Block Scope v Function Scope

27.1 let and const are block scope.

27.2 var is a function scope.

27.3 Block Scope refers to the visibility and lifetime of variables and functions within a block of code, which is typically defined by a pair of curly braces. Block scope was introduced in ECMAScript 6 (ES6) with the let and const keywords. Variables defined with the let and const are block-scoped and are accessible only within the block they are defined in.

27.4 Function Scope refers to the visibility and lifetime of variables and functions within the enclosing function. Variables declared with the var keyword have function scope and are accessible throughout the entire function, regardless of where they are defined within the function.

27.5 Block scope allows for better encapsulation and reduces the likelihood of variable collisions and unintended access.

28. Default parameters

28.1 Default parameters allow you to assign default values to function parameters if no argument or an undefined value is passed when calling the function. Default parameters were introduced in ECMAScript 2015 (ES6) and are a convenient way to handle missing or undefined values.

28.2 It is worth noting that default parameters are evaluated at the time the function is called, no when it's defined.

28.3 This allows you to use variables or expressions as default values. However, if a default parameter depends on the value of another parameter, you need to be cautious about the order in which the parameters are defined.

28.4 Additionally, default parameters work with any function, not just named function. They can be used with arrow functions, methods, and even function expressions.

29. Rest parameters

29.1 Rest parameters allow you to present an indefinite number of arguments as an array with a function. Rest parameter were introduced in ECMAScript 2015 (ES6) and provide a concise way to work with variable-length argument lists.

29.2 They are useful when you want to work with functions that can accept a variable number of arguments. Instead of accessing the argument through the 'arguments' object, you can use the rest parameter to gather them conveniently into an array.

29.3 They must be the last parameter in the function's parameter list, as it collects all remaining arguments. If desired, you can have regular parameters before the rest parameter.

29.4 The spread syntax can be used to pass an array to a function as separate argument. This is the inverse of rest parameter, where the spread syntax spread an array into individual arguments.

30. Parameter Destructuring

30.1 Parameter Destructuring allows you to extract individual values from an object or an array and assign them to variables within the function parameter list. It provides a concise way to access specific properties or elements of complex data structures.

30.2 It is a powerful technique that allows you to access specific values from objects or arrays more conveniently within function parameters. It can help simplify code and make it more readable, especially when working with complex data structures.

31. Callback / Higher Order Functions

31.1 Higher-order functions can take other functions as arguments and/or return functions as their results. Essentially, they treat functions as values, enabling powerful functional programming patterns and abstraction.

31.2 A callback function is a function that is passed as an argument to another function and is invoked or called within that function. Callback functions are a fundamental concept in Javascript, especially in asynchronous programming and event handling.

31.3 They are often used in scenarios where an operation takes time to complete, such as making an asynchronous request or waiting for an event to occur. The callback function allows you to specify what action should be taken once the operation is finished or the event is triggered.

31.4 Asynchronous functions in javascript commonly use callbacks to handle the result or error of an operation.

31.5 They allow for more flexible and dynamic programming patterns, such as executing different actions depending on the outcome of a function or operation.

32. Function returning Functions / Higher Order Functions

32.1 In Javascript, functions can return other functions, which is a powerful concept known as "function returning functions" or "higher-order functions". This allows for the creation of dynamic and specialized functions based on the input or configuration provided.

32.2 It is useful when you need to create variations or customizations of a core functionality. They provide a way to encapsulate behavior and create specialized function without duplicating code.

32.3 It facilitates code reuse, and contribute to a more expressive and functional programming style in JavaScript.

33. Important Array Methods

33.1 forEach() Method

33.1.1 forEach() method is a built-in method available on arrays. It allows you to iterate over the elements of an array and perform a specified action for each element.

33.1.2 It takes a callback function as an argument, which will be executed for each element in the array. The callback function can take up to three parameters: "currentValue", "index" (optional) and "array"(optional).

33.1.3 It is important to note that the forEach() method does not return a value. It simply performs an action for each element of the array. If you need to perform operations that accumulate a result or return a new value, other array methods like "reduce()" might be more suitable.

33.1.4 Although forEach() method is a method available on arrays and it takes a callback function as an argument, it does not meet the criteria of a higher-order function.

33.1.5 In javascript, an anonymous function is a function that does not have a name identifier. Instead of assigning a name to a function, we can assign it directly to a variable, or pass it as an argument to another function.
