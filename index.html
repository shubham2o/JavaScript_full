2. Backward Compatible, not Forward Compatible

2.1 JavaScript is a Backward Compatible language, this means if an old feature which was developed or introduced, let say 15 years ago, will also be easily work and perform in today's updated javascript language.

2.2 But, javascript is not forward compatible, this means the new javascript feature, let say, which is developed or introduced today will not be able to work on the old un-updated web browser.

2.3 The solution to this problem are some tools such as Babel, etc.

2.4 Babel transpile or converts the modern javascript code (which is written in today's updated JavaScript language containing new features) to the un-updated browser's supporting old javascript language and old features.

3. defer attribute

3.1 The defer attribute is used in HTML script tags to defer the execution of a Javascript file until the HTML page has finished parsing.

3.2 This means that the script will only be executed after the HTML has been completed loaded, parsed, and displayed to the user. This is useful for optimizing page load times because it allows the HTML to be displayed to the user faster, and the javascript can execute in the background.

3.3 If you want to add script tag in the head section of the html page, then you should add defer to the script tag as it will be deferred until after the HTML has been fully loaded and parsed.

4. "use strict"

4.1 The "use strict" statement is a directive that enables stricter parsing and error handling in your javascript code. When used, it will enforce stricter rules for syntax, variable declaration, and other common mistakes.

4.2 Note that "use strict" must be placed at the top of your JavaScript file or function in order to take effect.

5. Conventions and Rules

5.1 Always start with the small letter and use camelCase or snake_case.

5.2 You cannot use spaces.

5.3 You can use only underscore or dollar symbol.

5.4 You cannot start with a number.

6. String

6.1 In javascript, a string is a sequence of characters enclosed in single quotes(' ') or double quotes(" ").

6.2 Strings are used to represent textual data and can contain letters, numbers, symbols, and whitespace.

6.3 Strings are immutable, which means that once a string is created, its value cannot be changed. However, you can create new strings based on existing ones using various string methods or concatenation.
